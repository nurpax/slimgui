# Repository Guidelines

## Project Structure & Module Organization
- `src/` contains C++ bindings, generated includes (`*.inl`), and the Python package in `src/slimgui/`.
- `src/slimgui/` includes `imgui/`, `implot/`, `integrations/`, and `slimgui_ext/` (typing stubs).
- `tests/` holds pytest tests.
- `gen/` has code generation and vendor update scripts.
- `docs/` is documentation source; `dist/` is generated site output.
- `docs/recipes/` autogenerated files, do NOT touch by hand.
- `example/` contains runnable demos.
- `build/` and `cache/` are build artifacts.

## Build, Test, and Development Commands
- Read `DEVELOPMENT.md` before changing build or documentation workflows; it is the canonical local workflow guide.
- `python3 -m venv .venv && . .venv/bin/activate` to set up a local virtualenv.
- `pip install nanobind==2.9.2 scikit-build-core click glfw pyopengl numpy requests toml` for dev dependencies.
- `bash full_build.sh` builds the native extension and generates typings.
- `python3 gen/check_apiref_docs.py` validates authored VitePress API markdown against `.pyi` and runtime modules.
- `pytest` runs the test suite.

## Coding Style & Naming Conventions
- Python 3.13 or later, so `list[str]`, `str | None` instead of `Optional[str]`, etc.
- Python: `ruff` line length is 120 (see `ruff.toml`).
- Keep API naming aligned with Dear ImGui; avoid inventing new concepts.
- When regenerating enums or binding sources, use `clang-format`.
- Don't split function calls on too many lines, favor slightly longer lines.

## Testing Guidelines
- Framework: pytest; configured test paths are `tests/` and `gen/`.
- Naming: `tests/test_*.py`.
- Add tests for new API surface or regressions; update examples when behavior changes.

## Commit & Pull Request Guidelines
- When making commits in this repo, include a one-line subject plus a short multi-line description in the commit message; avoid subject-only commits.
- When writing multi-line commit messages from the shell, prefer a real multi-line form such as a heredoc or multiple `-m` arguments with actual newlines; do not embed literal `\n` sequences in the commit text.
- Use `git mv` instead of `mv` to move files.

## Python code example based documentation
- Example slimgui examples under `example/doc_examples/*.py`, f.ex. `example/doc_examples/buttons.py`
- Examples describe "Recipes" on how to do something with Slimgui.  These get built into documentation as .md files and screenshots.  These will be generated into `docs/recipes`.
- `gen/recipes/screenshot_render.py` renders screenshots from these recipes.
- `example/doc_examples/meta.py` adds decorators that are used to declare "examples".
- `example/run_examples.py` is an interactive browser of these doc examples.
